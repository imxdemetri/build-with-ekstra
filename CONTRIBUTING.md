# Contributing

Thanks for contributing to the public Ekstra developer surface.

## Scope

This repository is for:

- public documentation
- starter apps
- reference integrations
- browser control-profile package work

It is not the place for private roadmap material or internal-only runtime workbench changes.

## Good Contributions

- clearer onboarding docs
- starter improvements
- public developer ergonomics
- fixes to browser-facing examples
- package documentation and type improvements

## Before Opening A PR

1. Make the change as narrowly as possible.
2. Keep public-facing language external and developer-oriented.
3. Be explicit about what is available now versus planned.
4. Avoid introducing internal-only assumptions into public docs.

## Pull Request Notes

- include the developer-facing impact
- include any verification you performed
- keep product claims honest and concrete
