FAQ test targets

Does the phone need a native app?

No. The current flow uses a browser controller page and the phone's built-in motion APIs.

Does flick-click go through the runtime?

Yes. This demo uses the real run_detection_pipeline plus run_surface_pack path before the browser clicks.

What is different on iPhone?

Safari typically requires the controller page to be served over HTTPS before motion permissions are available.
